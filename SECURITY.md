# Security Policy

## Reporting a vulnerability

Please report suspected vulnerabilities privately to the project maintainers. Do not open public
issues for security-sensitive problems.

Include:

- A clear description of the issue
- Reproduction steps or proof of concept
- Impact assessment
- Any suggested remediation

## Response goals

- Initial acknowledgement within 3 business days
- Triage and severity assessment as quickly as possible
- Coordinated disclosure after a fix is available

## Scope notes

- Secret leakage in logs, docs, workflows, or test fixtures is in scope.
- Unsafe workflow permissions, compromised third-party actions, and supply-chain weaknesses are in
  scope.
- Misleading claims that imply the project creates a native Windows audio device when it does not
  are treated as documentation correctness issues, not security vulnerabilities.
